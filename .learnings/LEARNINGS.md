## [LRN-20260415-001] correction

**Logged**: 2026-04-15T00:00:00+08:00
**Priority**: high
**Status**: pending
**Area**: docs

### Summary
When a user asks for a directory structure or structure diagram, use a true tree with explicit parent-child connectors, not an indented pseudo-list.

### Details
The user explicitly required "画出目录结构，并加上中文注释". An indented block with comments was not sufficient because it did not clearly express parent-child hierarchy. The correct default is a real tree structure using connectors such as `├──`, `│`, and `└──`, plus concise inline Chinese annotations. This applies both to repository structure and generated target-repo structure unless the user asks for another format.

### Suggested Action
For future requests involving directory structures, architecture trees, generated file layouts, or install output layouts:
- Default to a real tree diagram with connectors
- Label whether each path is source template, generated at install time, generated by OpenSpec CLI, or generated at runtime when relevant
- Do not rely on surrounding prose to explain hierarchy that the tree itself should show

## [LRN-20260415-002] correction

**Logged**: 2026-04-15T00:00:00+08:00
**Priority**: high
**Status**: pending
**Area**: docs

### Summary
Generated image assets should go into a dedicated project subdirectory, not the repository root.

### Details
When exporting documentation diagrams or generated visual assets, placing files directly in the project root creates noise and degrades repository organization. The correct default is to create or reuse a purpose-specific directory such as `docs/diagrams/`, `docs/assets/`, or another clearly-scoped location, then place all generated images there consistently.

### Suggested Action
For future diagram/image export tasks:
- Create a dedicated output directory before generating files
- Prefer `docs/diagrams/` for README/documentation sequence diagrams unless the user specifies another location
- Update README or related docs if the repository structure section should reflect the new asset directory
